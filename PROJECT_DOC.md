## 🏗 시스템 아키텍처

```
┌─────────────────────────────────────────────┐
│     Pipedream (외부 스케줄러)                 │
│     매일 10:00 PM UTC (= KST 07:00) 정확히   │
└──────────────────┬──────────────────────────┘
                   │ POST workflow_dispatch (GitHub API)
┌──────────────────▼──────────────────────────┐
│              GitHub Actions                  │
│         workflow_dispatch 수신 즉시 실행      │
└──────────────────┬──────────────────────────┘
                   │
          ┌────────▼────────┐
          │    main.py      │
          │  날짜 기반 인덱스 │──→ (오늘 - 시작일) % 30
          │  커리큘럼 선택   │──→ curriculum.py 참조
          │  메시지 포맷 생성 │──→ 단어·예문·문법
          └────────┬────────┘
                   │
          ┌────────▼────────┐
          │    kakao.py     │
          │  OAuth 토큰 갱신 │──→ refresh_token → access_token
          │  자동 토큰 갱신  │──→ 새 token → GH_PAT로 Secret 업데이트
          │  메시지 전송     │──→ 카카오톡 나와의 채팅
          └─────────────────┘
```

## 🐛 이슈 해결 이력

### Issue 2 — GitHub Actions cron 스케줄 지연 (44~71분)

| | |
|---|---|
| **증상** | 매일 KST 07:00 예약이지만 실제 알림은 07:44~08:11에 도착 |
| **원인** | UTC 22:00 피크 타임 큐 적체, 평균 59분 지연 발생 |
| **해결** | GitHub cron 제거 → Pipedream 외부 스케줄러가 매일 10:00 PM UTC에 `workflow_dispatch` API 호출 |
| **결과** | 지연 없이 KST 07:00 정각에 알림 수신 |

### Issue 1 — GH_PAT 권한 부족 (Actions 실행 불가)

| | |
|---|---|
| **증상** | PAT로 workflow_dispatch API 호출 시 `403 Resource not accessible` |
| **원인** | fine-grained PAT에 Actions write 권한 미포함 |
| **해결** | GitHub 웹에서 수동으로 Run workflow 실행 |
